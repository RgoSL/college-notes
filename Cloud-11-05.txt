Docker -> Hospedagem de Containers

O Docker é dependente de um sistema operacional, baseados no sistema Unix

Containeres são simulações de outros SOs, como um Container que representa um IOS
ou um Windows

Virtual Machine (VM) - Emulação Virtual com as mesmas capacidades da máquina física


Servidor, é o lado de hospedagem da aplicação que fazemos.
Geralmente o servidor é em sistemas Linux

Dentro do servidor vai a nossa aplicação, um site por exemplo
Uma requisição é passada para o SERVER por meio de uma porta, HTTP básico.

Para uma relação entre dois servidores para suporte de demanda, existe um Balanceador para balancear as 
execuções.

O Docker e seus containeres trabalham em uma taxa de segurança de no máximo 80% de ocupação antes de gerar um
novo container

Exemplos de Nuvem

- AWS
- Azure
- Google Cloud
- Alibaba

Lambda -> Conceito de programação do Serveless, desenvolver sem a preocupação com o servidor.Porque quem vai se
preocupar é o serviço da Nuvem
Apenas é necessário especificar os parametros -> Memória total dedicada, Sistema Operacional que vai rodar

Graalvm em caso de Java é interpretada de maneira nativa, por meio de um código, traduzido para bytecode, que é total e unicamente compativel com um SO.

Na lambda o servidor só é iniciado a partir das requisições, não precisando estar ativo 24/7
