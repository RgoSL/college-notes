# Listas

<br>

## 📌 Conceito de Listas

- Estrutura de dados para representar sequência de elementos.
- Elementos organizados segundo uma política de manipulação.
- Permite inserção de dados.
- Permite remoção de dados.
- Permite acesso estruturado aos elementos.

<br>

## 📌 Listas Encadeadas

- Estrutura linear de nós.
- Cada nó aponta para o próximo elemento.
- Ligação entre elementos por referência.
- Último elemento aponta para `null`.

### Ligação entre Elementos
- João → Maria → Geraldo → `null`
- Cada elemento referencia o próximo.
- `null` indica o fim da lista.

<br>

## ⚙️ Classe Exemplo — Aluno

### Classe Aluno
- Representa um elemento da lista.
- Estrutura baseada em referência para o próximo nó.

### Atributos
- `nome : String`
- `idade : int`
- `proximo : Aluno`

### Função do atributo `proximo`
- Referencia o próximo elemento da lista.
- Cria ligação entre os nós.

<br>

## 📌 Tipos de Listas

### Lista Simples Encadeada
- Elementos organizados em sequência.
- Cada nó possui ponteiro para o próximo.
- Último elemento aponta para `null`.

### Lista Duplamente Encadeada
- Cada nó possui duas referências.
- Referência para elemento anterior.
- Referência para próximo elemento.

### Lista Encadeada Circular
- Último nó aponta para o primeiro.
- Estrutura em ciclo.
- Não existe elemento `null`.

<br>

## 📌 Listas Especializadas

### Fila (Queue)
- Estrutura baseada em FIFO.
- **First In, First Out**.
- Primeiro elemento que entra é o primeiro a sair.
- Inserção na extremidade final.
- Remoção na extremidade inicial.

### Pilha (Stack)
- Estrutura baseada em LIFO.
- **Last In, First Out**.
- Último elemento que entra é o primeiro a sair.
- Operações realizadas no topo da pilha.

<br>

## 💡 Características das Listas Encadeadas

### Vantagens
- Uso eficiente de memória.
- Estrutura dinâmica.

### Desvantagens
- Maior complexidade de implementação.
- Necessidade de manipulação de ponteiros.

<br>

## ❗ Atenção

- Erros em ponteiros podem quebrar toda a estrutura.
- Sempre verificar referências `null`.
- Escolher o tipo de lista adequado para cada problema.
