# Arquiteturas de Software, MVC e Exclusão de Dados

<br>

## 📌 Arquiteturas de Software

### Microserviços

- Separação em conjuntos menores.
- Serviços independentes.
- Baixo acoplamento.
- Escalabilidade facilitada.
- Reutilização de código mais difícil.

### Hexagonal (Ports and Adapters)

- Separação entre domínio e infraestrutura.
- Independência de frameworks.
- Facilidade para testes.
- Uso de portas e adaptadores.

### MVC (Model-View-Controller)

- Padrão arquitetural tradicional.
- Separação entre dados, interface e controle.
- Muito utilizado em aplicações web.

### Monolito

- Aplicação única.
- Código centralizado.
- Alto acoplamento.
- Mais simples no início.
- Escalabilidade mais difícil.

<br>

## 📌 Padrão MVC

### Model (Modelo)

- Representa dados e regras de negócio.
- Entidades do sistema.
- Manipulação do estado da aplicação.

### View (Visualização)

- Interface com o usuário.
- Exibição dos dados.
- Interação visual.

### Controller (Controle)

- Recebe requisições.
- Processa entradas do usuário.
- Coordena Model e View.

<br>

## ⚙️ Funcionamento do MVC

- Usuário interage com a View.
- Controller recebe a requisição.
- Validação de dados:
  - Campos obrigatórios.
  - Regras de senha.
- Processamento:
  - Autenticação.
  - Regras de negócio.
- Atualização do Model.
- Retorno para a View (ex: tela principal).

<br>

## 💡 Observações

- Separação de responsabilidades melhora manutenção.
- Facilita testes unitários.
- Torna o sistema mais organizado.

<br>

## ❗ Atenção

- Evitar lógica de negócio na View.
- Controllers muito grandes indicam problema de design.
- Model não deve depender da interface.

<br>

## 📌 Tipos de Exclusão

### Exclusão Física

- Remoção permanente do banco.
- Uso de `DELETE`.
- Dados não recuperáveis facilmente.

### Exclusão Lógica

- Uso de campo booleano (ex: `ativo`).
- Registro permanece no banco.
- Permite recuperação futura.

<br>

## 💡 Observações

- Exclusão lógica é ideal para auditoria.
- Muito utilizada em sistemas críticos.

<br>

## ❗ Atenção

- Sempre filtrar registros ativos.
- Pode impactar performance.
- Necessário padronizar uso no sistema.
