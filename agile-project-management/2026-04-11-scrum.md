# Etapas da Metodologia Scrum

<br>

## 📌 Product Backlog

### Definição
- Lista de requisitos do sistema  
- Inclui requisitos funcionais, não funcionais e regras de negócio  
- Base para planejamento do projeto  

### IBL (Item de Backlog)
- Unidade de funcionalidade  
- Representa uma rotina do sistema  
- Descrito como ação do sistema  

### Padrão de Nomeação
- Verbo no infinitivo + complemento  
- Ex: cadastrar cliente  
- Ex: gerar pedido  
- Ex: emitir nota fiscal  

### CRUD (Cadastrar / Manter)
- Incluir registro  
- Alterar registro  
- Consultar registro  
- Excluir registro  

### Entrada de Dados
- Formulários  
- QR Code  
- Leitura óptica  
- Importação de dados  
- Entrada por voz  

### Processamento de Dados
- Transformação de dados  
- Filtros  
- Ordenação  
- Classificação  
- Cálculos  
- Totalização  
- Agrupamento  

### Saída de Informação
- Exibição em tela  
- Relatórios (PDF / impresso)  
- Exportação de dados  
- Envio por SMS  
- Saída por voz  

### Estimativa de Tempo
- Definida no Product Backlog  
- Reutilizada na Sprint  
- Baseada em:
  - Pontos de história  
  - Prazo original  
  - Revisão da sprint  

<br>

## 📌 User Story

### Definição
- Descrição técnica dos IBLs  
- Detalha o funcionamento das rotinas  
- Base para desenvolvimento  

### História do Usuário
- Copiada do Product Backlog  
- Descreve necessidade do usuário  
- Ex: eu gostaria de cadastrar cliente  

### Atributos
- Campos da entidade  
- Tipo de dado definido  
- Definição de obrigatoriedade  

### Exemplo de Atributos
- cod_cli int(6) pk obrigatório  
- nome_cli string(40) obrigatório  
- cpf_cli string(11) fk obrigatório  
- tel_cli int(15) opcional  

### Regras de Negócio
- Validações do sistema  
- Condições obrigatórias  
- Ex: CPF inválido não salva  

### Descrição Técnica
- Detalhamento da implementação  
- Comportamento da rotina  
- Regras aplicadas ao processo  

<br>

## ⚙️ Task List

### Definição
- Cronograma simplificado  
- Organização das atividades  

### Priorização
- Alta  
- Média  
- Baixa  

### Datas
- Período de fevereiro a outubro  
- Julho desconsiderado  

### Responsáveis
- Membro do grupo definido  
- Cada tarefa possui um responsável  

### Status
- Pendente  
- Em andamento  
- Concluído  

<br>

## ⚙️ Sprint

### Definição
- Ciclos curtos de entrega  
- Duração de 2 a 4 semanas  

### Objetivo
- Entregar partes do sistema  
- Agrupar IBLs  

### Planejamento
- Quantidade definida pela equipe  
- Alinhamento com responsável/professora  

### Relação com Product Backlog
- Utiliza os IBLs definidos  
- Utiliza estimativas de tempo  

### Sincronização
- Dados devem ser consistentes  
- Backlog, sprint e tarefas alinhados  

<br>

## 💡 Observações Importantes
- Product Backlog é dinâmico e pode mudar ao longo do projeto  
- User Stories devem ser simples e focadas no usuário  
- Sprints precisam ter escopo bem definido  
- Task List ajuda no controle operacional do time  

<br>

## ❗ Atenção
- Falta de clareza nos IBLs gera retrabalho  
- Estimativas mal feitas comprometem a sprint  
- Desalinhamento entre backlog e tarefas causa inconsistência  
- Excesso de detalhe em User Story pode atrapalhar agilidade
