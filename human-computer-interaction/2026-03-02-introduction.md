# Interação Humano-Computador (IHC) 

<br>

## 📌 Conceitos Principais

* Área que estuda o projeto, avaliação e implementação de sistemas interativos
* Foco nas pessoas e no uso dos sistemas
* Campo consolidado desde os anos 1980

### Objetivo

* Entender como as pessoas pensam
* Compreender tomada de decisões
* Analisar interação com sistemas
* Projetar sistemas eficientes, seguros e usáveis

### Design Centrado no Usuário

* Foco na interação do usuário com o produto
* Considera acessibilidade
* Interfaces seguem padrões conhecidos
* Expectativa de consistência entre sistemas

### Intuitividade

* Depende do público-alvo
* Varia com idade e experiência

<br>

## 📌 Tecnologia e Sociedade

* Tecnologia não é neutra

### Impactos

* Influencia comportamento humano
* Molda relações sociais
* Afeta cultura e economia

### Exemplo

* Redes sociais mudaram padrões de comunicação

* Em IHC, analisa-se como o design impacta o comportamento

## 📌 Conceitos Fundamentais

### Interação

* Comunicação entre usuário e sistema

### Interface

* Meio onde ocorre a interação

### Affordance

* Indica como algo deve ser usado
* Baseado na percepção
* Define clareza de uso

### Origem

* Conceito de James J. Gibson
* Popularizado por Donald Norman

### Exemplos

* Botão elevado → parece clicável

* Barra de rolagem → indica movimento

* Ícone de lixeira → sugere exclusão

* Interface confusa indica falha de affordance

<br>

## 📌 Qualidade em IHC

* Qualidade não é só funcionamento

### ISO 9241

* Ergonomia da interação humano-sistema

### Critérios

* Eficácia → concluir tarefa
* Eficiência → esforço necessário
* Satisfação → experiência do usuário

<br>

## ⚙️ Teoria da Atividade

* Comportamento mediado por ferramentas
* Interação ocorre em contexto social

### Estrutura

* Usuário → usa → ferramenta → atinge objetivo

### Contexto Social

* Regras
* Cultura
* Comunidade
* Divisão de tarefas

<br>

## 💡 Observações Importantes

* Interfaces devem reduzir esforço cognitivo
* Padrões conhecidos aumentam usabilidade
* Usuário é o centro do sistema

<br>

## ❗ Atenção

* Interface bonita ≠ interface usável
* Ignorar contexto do usuário gera falhas
* Excesso de funcionalidades prejudica a experiência
